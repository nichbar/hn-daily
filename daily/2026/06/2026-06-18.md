# 本期热点

今天 Hacker News 最受关注的话题，是瑞士联邦议会以微弱多数投票通过了解除新建核电站禁令的议案。这一决定标志着瑞士自 2017 年公投确立的弃核路线出现重大转向，也引发了关于能源安全与气候目标之间权衡的广泛讨论。与此同时，科技领域的几条新闻同样分量十足：独立安全研究员发现 GitHub 上潜伏着上万条分发木马的仓库；AMD 在固件更新中静默移除了消费级 CPU 的内存加密功能；微软新 Outlook 与经典版之间的性能差距被量化曝光。本期内容还涵盖了 Craigslist 创始人低调捐出半数身家的故事、北欧零售巨头因强制同意被罚的 GDPR 案例、老药新用的低成本创新体系，以及阿尔伯塔省长达七十余年的灭鼠奇迹。

## 政策与治理

### 瑞士议会解除新建核电站禁令，最终由全民公投决定

[瑞士联邦议会](https://www.bluewin.ch/en/news/switzerland/parliament-lifts-ban-on-new-nuclear-power-plants-3257535.html)国民院以 100 票对 98 票的微弱优势，投票通过了解除新建核电站禁令的反提案。该议案此前已获得联邦院和联邦委员会支持，最终将由全民公投决定生效与否。支持者认为，在全面电气化转型和冬季水电产能不足的背景下，核能是不可或缺的基荷电源；反对者则来自左翼和绿党阵营，担忧安全与核废料处理问题。

HN 社区的讨论集中于瑞士的特殊国情。有当地用户指出，瑞士面临冬夏能源不平衡的突出矛盾——夏季冰川融水和大坝蓄水充沛，冬季则高度依赖从法国进口核电。"我们不能一直买法国的核电，"一位瑞士评论者写道，"即便要建造新电站，也需要非常审慎地评估。"另有观点认为，2017 年瑞士首次禁核公投深受德国福岛后弃核决定的影响，而如今风向转变本身就是一种理性回调。不过，多位评论者提醒，即便解禁，新电站从审批到实际发电仍需十年以上周期。

### 欧洲机构迁移至 W Social，数字主权口号与实际行动背道而驰

调查记者 Elena Rossini 在[博客文章](https://blog.elenarossini.com/w-social-public-institutions-and-the-theater-of-european-digital-sovereignty/)中披露，欧洲委员会主席 Ursula von der Leyen、欧洲央行行长 Christine Lagarde 等欧盟核心机构的 Bluesky 账户已被迁移至 W Social。W Social 是一家瑞典创业公司运营的社交平台，主打"欧洲数字主权"概念，要求用户进行身份验证以防止机器人和虚假信息。但调查发现，W Social 已于今年 3 月将 GitHub 仓库转为私有状态——这意味着代码不再公开。Rossini 在文章中指出，这一做法与欧盟在 6 月 3 日公布的以开源为核心的"技术主权一揽子计划"背道而驰。

HN 评论区对这些迁移决策持强烈质疑态度。多位读者发问："为什么不是 Eurosky？"Eurosky 是一个由非营利基金会运营、开发过程完全透明的 atproto 社交网络，被视为更符合欧洲数字主权精神的选项。高赞评论提醒，用户们一次又一次跳入由营利公司控制的社交平台，最终都沦为被出售的产品。有技术背景的读者指出，atproto 协议虽然允许用户自由迁移 Personal Data Server，但如果 relay 和 appview 层被封闭，所谓的"迁移自由"可能只是空谈。

### 五年前的一封投诉邮件，最终让北欧零售商付出 180 万欧元代价

一位 GDPR 专家在个人[博客](https://www.thatprivacyguy.com/blog/elkjop-forced-consent-fine/)中讲述了长达五年的维权历程。2021 年，他发现北欧零售集团 Elkjop 的会员制度存在设计缺陷：消费者要停止营销邮件就必须取消会员资格，这实际上将"同意"与会员权益捆绑。他致信公司数据保护官指出这一安排违反 GDPR，并投诉至瑞典监管机构。历经案件从瑞典移交挪威的波折后，挪威数据保护局于本月对 Elkjop 开出 2000 万挪威克朗（约 180 万欧元）的罚单，认定该会员制度下的"同意"并非自由给予。更具讽刺意味的是，投诉人本人是通过志愿者运营的维基才得知裁决结果，而非来自监管机构的正式通知。

"强制同意、付款即同意、捆绑同意——整个'不同意就无法使用服务'的模式无处不在，而且每次都是因为同一个简单原因而不合法，"原文作者写道。HN 评论区对此事的反响明显两极。有评论者佩服投诉人的坚持，但指出在美国"行使权利会被视为制造麻烦"。也有用户直言，"如果我在欧盟做生意，我会把这个客户拉黑——他对公司来说风险太大了。"这一观点在评论区遭到多人反驳，其中一位写道："因为客户懂法律就害怕？那是彻头彻尾的失败者心态。"

## 技术与产品

### GitHub 被曝存在上万木马分发仓库，开源安全信任基础遭动摇

独立安全研究员在[个人博客](https://orchidfiles.com/github-repositories-distributing-malware/)中披露，GitHub 上存在约 10000 个分发木马恶意软件的仓库。这些仓库伪装成无害的开源项目，共享一个模式：每隔数小时删除旧提交并推送新提交，仅在 README 中添加指向压缩包的外部链接。压缩包内含可执行文件， VirusTotal 检测为木马。研究员利用 GHArchive 事件流和 GitHub API 编写检测脚本，发现这批仓库已存在数月甚至超过一年，而 GitHub 的安全机制始终未能自动识别。文章发布后，GitHub 开始批量删除这些仓库。

"开源代码可视即可信"这一假设，在 HN 讨论中被多位用户指出早已不成立。有评论者引用自己的开发经历说明：大多数用户从不会审计所安装软件的源代码，更不会验证二进制文件与源代码的一致性。一位开发者分享了他最近纠结是否将 H2 数据库以 jar-in-jar 方式打包到模组中，最终因为无法确保间接依赖安全而放弃的经历。讨论中还提到，2019 年以来类似的 GitHub 投毒事件多次被曝光，但平台层面的自动化检测一直存在盲区。一位资深用户总结道："GitHub 已经成为软件分发的中心，但用星标数量来判断安全性的做法太过天真。"

### 微软新 Outlook 加载邮件比经典版慢十倍

[Windows Latest](https://www.windowslatest.com/2026/06/15/microsofts-new-outlook-takes-10-seconds-to-do-what-outlook-classic-does-instantly-on-windows/) 的对比测试显示，基于 WebView2 的新版 Outlook 在点击通知后加载一封邮件需要约 10 秒，而经典版 Outlook 几乎是瞬间完成。新版本质上是一个 Web 应用，每次交互都需要大量渲染和数据加载，经典版作为原生应用则可以直接访问本地存储。微软一直在推动用户从经典版迁移至新版，这一性能差距让许多用户感到不满。

不少 HN 评论者指出，问题根源不在 Web 技术本身。有人提到 Fastmail 的 Web 客户端同样基于浏览器技术，响应速度却不亚于原生应用。多位读者质疑，微软在 AI 辅助编程工具 Copilot 上投入巨大，却连邮件客户端的基本性能都无法保证。"微软的问题在于组织文化，"一位自称微软工程师的用户写道，"开发者总是被要求交付更多功能，根本没有时间做正确的事。"也有反对声音认为，新 Outlook 跨平台且无广告，对 Linux 用户而言已经是难得的改善。

### AMD 在固件更新中静默移除 Ryzen 内存加密

[Tom's Hardware](https://www.tomshardware.com/pc-components/cpus/amd-silently-removes-memory-encryption-from-consumer-ryzen-cpus-leaving-users-unaware-that-they-may-be-vulnerable-security-feature-vanishes-after-newer-agesa-firmware-amd-engineers-go-radio-silent-when-pressed-about-the-change) 报道，AMD 在最新的 AGESA 固件更新中静默移除了消费级 Ryzen 处理器的内存加密功能。该功能从未被列为正式特性，但在过去几代 Ryzen CPU 中实际存在，可用于防御冷启动等物理访问攻击。AMD 工程师在被追问此事时保持沉默。

HN 社区对此事的看法明显分裂。一部分用户认为，内存加密从未被官方文档化或支持，AMD 没有义务维护未公开功能。"如果我用 REST API 的未文档化端点，被改了也不能抱怨。"另一部分用户则将其视为"售后降级"——厂商在用户完全不知情的情况下通过固件削弱产品能力。有服务器运维背景的读者指出，消费级 Ryzen 在小型企业和家庭实验室中被广泛使用，对这些用户而言内存加密确有实际价值。一些评论者还担心，这一先例可能为未来更多的售后功能移除打开大门。

## 科学与研究

### 医院和大学以降低 90% 成本的方式开展老药新用试验

[伦敦国王学院](https://www.kcl.ac.uk/news/hospitals-and-universities-repurposing-drugs-at-90-lower-cost)的研究团队在《剑桥法律杂志》上发表论文，揭示了一个"隐藏"的药物创新体系：医院和大学正在以比制药行业低 90% 的成本进行老药新用的后期临床试验。这套体系运行于专利保护之外，已产出了多个令人振奋的案例——将抗癌药物用于治疗致盲性眼病、将乳腺癌治疗药物转为预防用途、使用老牌抗炎药治疗新冠。研究者认为，这种平行创新体系有潜力为社会持续提供可负担的治疗方案。

"药价问题在美国尤其突出，"HN 讨论区中有读者指出。一位评论者用数据说明：美国品牌药价格是其他国家的 4 倍以上，但非专利仿制药却比其他国家便宜三分之一——原因在于品牌药受专利保护形成人为垄断，而非自由市场运作的结果。也有读者将老药新用的模式类比为开源软件社区——去中心化、低成本、成果共享。不过，讨论中也有冷静的声音：这种模式目前仍然停留在个案层面，是否可规模化推广还有待验证。

### 阿尔伯塔如何成为地球上最大的无鼠区域

[Works in Progress](https://worksinprogress.co/issue/albertas-war-on-rats/) 刊发深度报告，讲述加拿大阿尔伯塔省长达七十余年的灭鼠故事。阿尔伯塔省面积约 50 万平方公里，是除南极洲外全球最大的无鼠区域。1950 年，褐家鼠从东部萨斯喀彻温省侵入阿尔伯塔，省政府当机立断在边境设立约 600 公里长的"鼠控区"，集中资源进行监测、投毒和公众教育。早期使用了 63 吨三氧化二砷追踪粉，后期改用更安全的抗凝血剂华法林——监管人员甚至在社区会议上当众食用含华法林的燕麦片以证明其安全性。如今，阿尔伯塔每年仅投入约 50 万加元用于持续监控，2025 年收到的 875 起鼠情报告中仅 47 起被确认为真鼠。

多位加拿大用户分享了自己的亲身经历。"我在阿尔伯塔生活了几十年，从没见过老鼠，也没见过蟑螂。"一位 HN 读者写道，"但开车往东一个小时，情况就完全不同了。"也有评论者澄清，"无鼠"并不意味着完全没有老鼠进入——老鼠一直在从周边省份渗透，只是通过持续监控始终无法建立繁殖种群。这与纽约市每年花费 470 万美元却收效甚微的鼠患治理形成了鲜明对比。讨论中诞生了一个有趣的观察：拥有海港的城市因为历史上已建立稳定的鼠群而难以根除，而阿尔伯塔没有海港，鼠类只能通过陆路进入——这一地理特征为前置防御创造了条件。

## 社会与文化

### Craigslist 创始人捐出 5 亿美元，与科技富豪的"吝啬浪潮"形成对照

[The Independent](https://www.independent.co.uk/us/money/craigslist-multimillionaire-craig-newmark-b2980681.html) 的人物报道讲述了 Craigslist 创始人 Craig Newmark 的故事。这位 74 岁的前软件工程师已通过个人慈善基金会捐出 5 亿美元，支持网络安全、新闻业、军人家庭和鸽子救助等事业。他在 2025 年加入"捐赠誓言"（Giving Pledge），承诺捐出大部分财富。然而，Peter Thiel 等科技富豪正积极劝说同侪退出该誓言，称捐款将流向"左翼非营利组织"。Newmark 在《纽约时报》上撰文回应："有太多钱了，所以我把它给出去。我无法理解那些批评这种做法的人。"报道描绘了一个与硅谷奢华文化格格不入的形象：这个坐拥数亿身家的人不开车、乘坐公共交通、每天喂养两只鸽子。

HN 社区在肯定 Newmark 的同时，也指出了 Craigslist 平台长期存在的诈骗问题。多位用户反映大约四分之一的租房帖是骗局，平台从未采取有效措施。"Craigslist 不向租房帖收费，也就没有动力去治理欺诈，"一位读者写道。另有评论者指出 Newmark 的个人善举与平台不作为之间并不矛盾——公司的日常运营由管理团队负责，创始人的个人价值观不一定转化为商业策略。不过，大多数评论者认同 Newmark 与 Thiel 之间的对立是硅谷内部关于财富伦理的一次清晰分野。

### 美国工资跑不赢房租，经典的"30% 规则"已失效

[Realtor.com](https://www.realtor.com/advice/rent/wages-in-america-are-too-low-for-the-30-rule-to-work-for-renters-anymore/) 撰文指出，"房租不超过收入 30%"这一经典的 affordability 准则在当今美国已基本失效。该规则基于税前总收入，但扣除退休金、税款和健康保险后，美国家庭税后收入大幅缩水。以家庭收入中位数 84000 美元计算，中位数租金 1686 美元占税后收入的比例高达 53%，远超 30%。为了应对住房成本压力，58% 的美国年轻人在搬出父母家后又搬回，65 岁以上老年人则成为增长最快的合租群体。

HN 评论区的核心批判指向问题的根源——住房供给不足，而非工资增长不够。多位读者指出，Realtor.com 作为房地产行业组织，有动机将讨论引向"涨工资"而非"降房价"。有用户一针见血地写道："供需关系，就是这么简单。"在严格限制土地供应的城市，增加工资只会进一步推高房价。也有读者分享了自己的生存策略——刻意接受超过 30% 的房租比，以换取更短的通勤时间。不过，也有观点认为，使用 50/30/20 预算法则（将全部生活必需品支出控制在税后收入的 50% 以内）比单一的 30% 房租规则更能反映真实的财务健康状况。

# 尾巴

今天的 Hacker News 讨论涵盖了从能源政策到开源安全、从消费者权益到住房危机的广阔议题。瑞士核电路线的转向提醒我们，气候目标与能源安全之间的张力正在重塑各国的政策选择；上万木马仓库的曝光则再次叩问：在软件依赖链日益复杂的今天，"信任"意味着什么。Craigslist 创始人的朴素捐赠与 Elkjop 的 GDPR 罚单，从不同侧面折射出财富与权利的责任。感谢阅读，我们下期再见。
