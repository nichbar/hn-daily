# 本期导读

今天的技术圈充满了戏剧性：从微软内部对 Claude Code 的“倒戈”，到 OpenAI 推出的多代理协作中心，再到马斯克将算力推向太空的宏大愿景，AI 正在以前所未有的速度重塑生产力工具。同时，Sudo 维护者长达 30 年的坚守也引发了我们对开源可持续性的深思。

## 资讯

### [Claude Code](https://www.theverge.com/tech/865689/microsoft-claude-code-anthropic-partnership-notepad) 迅速占领微软内部

尽管微软正全力推销自家的 GitHub Copilot，但其实验室内部却传出了不同的声音。消息称，包括 Windows 和 Office 在内的核心团队正被鼓励使用 Anthropic 的 [Claude Code](https://www.theverge.com/tech/865689/microsoft-claude-code-anthropic-partnership-notepad) 进行开发和原型设计。微软目前采取了双轨制，要求工程师对比两者的优劣。社区对此反响热烈，不少用户指出，内部员工的选择往往是产品质量最诚实的反馈，而微软复杂的命名传统也再次成为了吐槽的焦点。

### OpenAI 发布 [Codex](https://openai.com/index/introducing-the-codex-app/) 桌面应用：多代理协作指挥中心

OpenAI 为 macOS 用户带来了一款全新的生产力工具。这款名为 [Codex](https://openai.com/index/introducing-the-codex-app/) 的应用支持多代理并行工作，并通过“工作树”功能有效管理不同任务的代码副本。除了编程，它还整合了设计、项目管理和自动化部署等技能，试图将 AI 变成全职的数字助理。虽然 Electron 框架带来的内存占用引发了一些争议，但其构建新一代 AI 应用生态的野心已清晰可见。

### [Sudo](https://www.millert.dev/) 维护者：30 年如一日的开源守护者

[Todd C. Miller](https://www.millert.dev/) 坚守 [sudo](https://www.millert.dev/) 维护岗位已逾 30 载。作为支撑全球基础设施的安全支柱，Todd 如今却需要寻找赞助以资助项目的持续开发。他在个人网站低调寻求支持的消息激起了社区对“开源白嫖”现状的强烈声讨。许多开发者认为，大公司对核心工具的无偿消费是不可持续的，关于建立收费许可或转向 Rust 版 sudo-rs 的讨论也再次升温。

### [Anki](https://forums.ankiweb.net/t/ankis-growing-up/68610) 易主：19 载后交棒 AnkiHub

知名背诵工具 [Anki](https://forums.ankiweb.net/t/ankis-growing-up/68610) 的创始人 Damien Elmes 宣布将项目移交给 AnkiHub 团队。在独立维护 19 年后，Damien 选择以此方式减轻压力并促进项目可持续发展。尽管 AnkiHub 承诺保持开源，但社区仍对潜在的商业化和“劣币化”表示担忧，尤其是关于未来云服务是否收费的问题成为了讨论焦点。

### [Waymo](https://www.bloomberg.com/news/articles/2026-01-31/waymo-seeking-about-16-billion-near-110-billion-valuation) 拟以 1100 亿美元估值再融资 160 亿美元

自动驾驶领头羊 [Waymo](https://www.bloomberg.com/news/articles/2026-01-31/waymo-seeking-about-16-billion-near-110-billion-valuation) 正在寻求新一轮巨额融资，其日订单量已突破 100 万大关。用户普遍认为其驾驶表现比人类更平顺、更安全。本轮资金预计将用于扩大车队规模并进入更多城市。分析认为，Waymo 正在向外界证明 Robotaxi 商业模式在大规模运营下的可行性。

### [xAI](https://www.spacex.com/updates#xai-joins-spacex) 并入 SpaceX：迈向卡尔达舍夫 II 级文明的星际计算网

SpaceX 宣布收购 xAI，旨在将 AI 算力与航天运力深度结合。埃隆·马斯克表示，地球的资源已不足以支撑 AI 的指数增长，利用星舰运力构建“空间数据中心”是迈向戴森球原型的第一步。尽管散热和碎片管理仍存物理挑战，但这一科幻级的愿景确实为解决算力瓶颈提供了全新的宏大思路。

## 博客

### 苹果 [MacBook Pro](https://lapcatsoftware.com/articles/2026/2/1.html) DFU 端口文档存在错误

一位开发者在尝试更新外置磁盘上的 macOS 时发现，[苹果官方文档](https://lapcatsoftware.com/articles/2026/2/1.html) 关于 DFU 端口位置的描述与实际不符。这一低级错误导致了多次更新失败。作者由此批评了 Apple Silicon 时代繁琐的磁盘管理策略和系统报错信息模糊的 UX 缺陷，引发了开发者对苹果硬件文档质量的广泛讨论。

### [Nano-vLLM](https://neutree.ai/blog/nano-vllm-part-1)：揭秘推理引擎的工作原理

[Nano-vLLM](https://neutree.ai/blog/nano-vllm-part-1) 通过约 1200 行 Python 代码拆解了大语言模型推理引擎的核心机制。文章深入浅出地介绍了 Block Manager、PagedAttention 以及显存优化的底层逻辑。这类高质量的技术拆解能有效帮助开发者理解复杂系统的底层逻辑，被誉为 AI 时代极具价值的技术科普。

### 黑掉 [Moltbook](https://www.wiz.io/blog/exposed-moltbook-database-reveals-millions-of-api-keys)：“氛围编程”的安全隐患

Wiz 研究团队披露了一个由 AI 辅助构建的社交网络 [Moltbook](https://www.wiz.io/blog/exposed-moltbook-database-reveals-millions-of-api-keys) 的重大漏洞。由于完全依赖 Prompt 生成代码而缺乏安全审核，该平台的数据库配置错误导致百万级 API 密钥完全暴露。这一案例为盲目推崇“氛围编程”（Vibe Coding）的趋势敲响了警钟：降低门槛的同时，绝不能忽视基础的安全规范。
